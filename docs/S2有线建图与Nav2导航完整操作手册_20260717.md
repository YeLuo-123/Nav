# S2 有线建图与 Nav2 导航完整操作手册

## 1. 交付内容和职责

本交付包用于另一台笔记本通过网线直接连接 S2 移动底盘，完成：

- 接收前后 Livox 雷达融合点云和底盘里程计。
- 按高度切片、射线更新和噪点过滤，实时生成二维占据图。
- 在 RViz 同时显示三维点云、二维地图、S2 STL 模型和碰撞轮廓。
- 使用 Navigation2 进行路径规划、局部避障和小范围实机移动。

当前二维地图算法是“`/controller/odom` 约束下的点云占据栅格融合”，不是具有
回环检测的完整 SLAM。短距离测试可用，长距离里程计漂移会造成地图重影。

## 2. 硬件和系统要求

- Ubuntu 22.04 x86_64 笔记本。
- ROS 2 Humble Desktop，安装在 `/opt/ros/humble`。
- 千兆以太网接口和可靠网线。
- S2 底盘控制器地址 `192.168.127.10`。
- 笔记本网口地址 `192.168.127.100/24`。
- `ROS_DOMAIN_ID=0`，`rmw_cyclonedds_cpp`。

不要把 Conda 环境用于 ROS 节点。本包统一使用 `/usr/bin/python3`。

## 3. 首次安装

将整个交付文件夹放到任意位置，不要只复制其中的 `tools` 子目录。进入目录：

```bash
cd /你的路径/S2激光雷达建图与Nav2导航交付包_20260717
chmod +x bin/* setup/*.sh tools/*.sh tools/*.py
./setup/install_and_build.sh
```

安装脚本会：

1. 安装 Python、CycloneDDS、RViz 和 robot_state_publisher 依赖。
2. 解压本包携带的 Navigation2 Humble 固定版本源码。
3. 使用 `rosdep` 安装依赖。
4. 在本包的 `nav2_ws` 中源码编译 Nav2。

需要网络和 sudo 权限。成功标志：

```text
[s2-install] build complete: .../nav2_ws/install/setup.bash
```

## 4. 每次有线测试前配置

插入网线并查看接口名：

```bash
ip -br link
```

常见名称是 `eno1`、`enp3s0` 或 USB 网卡的 `enx...`。以下示例假设为
`enp3s0`，每个新终端都先设置：

```bash
cd /你的路径/S2激光雷达建图与Nav2导航交付包_20260717
export S2_NETWORK_INTERFACE=enp3s0
```

终端 1，配置固定 IP：

```bash
./bin/s2_mapping network
ping -c 3 192.168.127.10
```

`network` 会创建 NetworkManager 连接 `DREAM-S2-test`，不会把该网口设为默认
互联网路由。

## 5. 检查雷达、里程计和软件环境

```bash
./bin/s2_doctor
```

全部显示 `[OK]` 才进入导航测试。重点是：

- `/driver/lidar/point_cloud/Data`：约 10 Hz，约 40032 点/帧。
- `/controller/odom`：约 200 Hz。
- 网卡 carrier、固定 IP 和控制器 ping 正常。
- `nav2_ws/install/setup.bash` 存在。

需要生成详细 ROS 话题报告时：

```bash
./bin/s2_mapping diagnose
```

## 6. 启动实时二维建图

终端 1：

```bash
cd /你的路径/S2激光雷达建图与Nav2导航交付包_20260717
export S2_NETWORK_INTERFACE=enp3s0

export S2_LIDAR_MIN_Z=0.25
export S2_LIDAR_MAX_Z=1.20
export S2_LIDAR_MAX_RANGE_M=8.0
export S2_MAX_CLOUD_ODOM_DELTA_SEC=0.05

./bin/s2_mapping start
```

每次 `start` 自动创建新的 `output/s2_时间戳`，避免旧图残留。查看状态：

```bash
./bin/s2_mapping status
```

通过标准：

- `mapper_running=true`、`http_running=true`。
- `pose_valid=True`。
- `cloud_count`、`integrated_scan_count` 持续增加。
- `sync_skip_count` 接近 0。
- `map_file_age_sec` 接近 0。

输出位置：

```bash
./bin/s2_mapping files
```

主要文件：

- `lidar_nav_map_latest.pgm`：导航占据图。
- `lidar_nav_map_latest.yaml`：分辨率、原点和 PGM 引用。
- `lidar_nav_map_latest.png`：人工查看预览。
- `lidar_nav_map_latest.json`：机器人位姿、计数和同步状态。
- `mapper.log`：建图日志。

ROS 输出：

- `/s2_lidar_slam/map`
- `/s2_lidar_slam/point_cloud`

## 7. 打开 RViz

终端 2：

```bash
cd /你的路径/S2激光雷达建图与Nav2导航交付包_20260717
export S2_NETWORK_INTERFACE=enp3s0
./bin/s2_rviz
```

RViz 中可见：

- 三维融合雷达点云。
- 实时二维占据地图。
- 来自 `robot_description` 的真实 S2 STL 模型。
- 黄色 `0.92 x 0.72 m` 碰撞轮廓。
- `odom -> base_link` 实时 TF。

点云遮挡模型时，临时关闭 `S2 Fused PointCloud`。检查导航安全余量时打开
`Configured Collision Footprint`、全局和局部 costmap。

## 8. Nav2 只规划测试

保持建图运行，关闭单独 RViz，终端 2 执行：

```bash
cd /你的路径/S2激光雷达建图与Nav2导航交付包_20260717
export S2_NETWORK_INTERFACE=enp3s0
./bin/s2_nav2 preview
```

`preview` 启动完整 Nav2 和 RViz，但速度只发布到预览话题，不连接底盘控制。
在 RViz 使用 `Nav2 Goal` 选点，检查全局路径和局部路径能否生成。

预览话题：

```bash
ros2 topic echo /s2_nav2/cmd_vel_preview
```

## 9. 小范围实机导航

先按急停、清空周围人员和杂物，确认地图没有明显重影。终端 2：

```bash
cd /你的路径/S2激光雷达建图与Nav2导航交付包_20260717
export S2_NETWORK_INTERFACE=enp3s0
./bin/s2_nav2 real
```

`real` 模式会：

1. 把底盘切换到软手动模式。
2. 退出驻车。
3. 将 Nav2 速度限制为 `x=0.12 m/s`、`y=0.10 m/s`、`yaw=0.30 rad/s`。
4. 通过 `ws://192.168.127.10:6001/api/AMR/ManualMove` 发送速度。
5. 退出时发送零速度并重新驻车。

首次只在明确白色自由区选择 `0.3-0.5 m` 目标。急停必须在手边。按
`Ctrl+C` 正常退出，或在 Navigation 2 面板 Cancel。

## 10. 停止建图

```bash
./bin/s2_mapping stop
```

停止后当前会话的 PGM、YAML、PNG 和 JSON 会保留。

## 11. 常用参数

```bash
export S2_NETWORK_INTERFACE=enp3s0
export S2_CONTROLLER_IP=192.168.127.10
export S2_HOST_CIDR=192.168.127.100/24
export S2_ROS_DOMAIN_ID=0
export S2_LIDAR_MIN_Z=0.25
export S2_LIDAR_MAX_Z=1.20
export S2_LIDAR_MAX_RANGE_M=8.0
export S2_LIDAR_HTTP_PORT=8776
```

Nav2 参数位于 `configs/navigation/s2_nav2_params.yaml`。当前底盘 footprint 为
`0.86 x 0.66 m`，另加 `0.03 m` padding；障碍膨胀半径为 `0.25 m`。

## 12. 常见问题

### 找不到底盘或 ROS 话题

- 检查 `ip -br link` 和网线 carrier。
- 确认网卡名与 `S2_NETWORK_INTERFACE` 一致。
- 重新执行 `./bin/s2_mapping network`。
- 检查 `ping 192.168.127.10`。

### 地图移动后出现重影

当前算法依赖 `/controller/odom`，里程计跳变或漂移会把同一障碍积分到多个位置。
停止旧会话并重新 `start`。不要继续在已污染的 log-odds 地图上测试导航。

### 可通行区域显示未知

雷达只有射线实际扫过的区域才会标记为已知自由区。需要让机器人完整扫过任务区域，
不能把灰色未知区当作安全空间。

### RViz 模型不显示

检查：

```bash
ros2 topic info /robot_description
ros2 topic hz /joint_states
ros2 run tf2_ros tf2_echo odom base_link
```

并确认整个 `robot_description/meshes` 目录没有漏复制。

### Nav2 能规划但底盘不动

`preview` 本来就不驱动底盘。实机测试使用 `./bin/s2_nav2 real`，同时检查急停、
驻车和控制器 HTTP/WebSocket 接口。
